### 6.2 Provider 基类设计

```python
from abc import ABC, abstractmethod
from dataclasses import dataclass
from typing import Optional, List

@dataclass
class BalanceInfo:
    """按量付费余额信息"""
    balance: float              # 余额（元）
    currency: str = "CNY"       # 货币
    error: Optional[str] = None # 错误信息

@dataclass
class PlanInfo:
    """套餐额度信息"""
    name: str                   # 套餐名称（如 "Coding Plan Lite"）
    plan_type: str              # 类型：coding_plan / token_plan / agent_plan
    used: float                 # 已用量
    total: float                # 总量
    unit: str                   # 单位（万Token/Credits/AFP等）
    expire_date: Optional[str] = None  # 到期日期
    error: Optional[str] = None

@dataclass
class ProviderStatus:
    """厂商完整状态"""
    name: str                   # 显示名称
    icon_color: str             # 主题色
    auth_type: str              # api_key / cookie
    balance: Optional[BalanceInfo] = None
    plans: List[PlanInfo] = None
    configured: bool = False    # 是否已配置

class BaseProvider(ABC):
    """厂商适配器基类"""
    
    name: str = ""              # 厂商名称
    icon_color: str = "#888888" # 主题色
    auth_type: str = "api_key"  # 认证类型
    
    @abstractmethod
    def get_balance(self, config: dict) -> BalanceInfo:
        """
        查询按量付费余额
        
        Args:
            config: 配置信息，包含 api_key 或 cookie
            
        Returns:
            BalanceInfo: 余额信息
        """
        pass
    
    def get_plans(self, config: dict) -> List[PlanInfo]:
        """
        查询套餐额度（可选实现）
        
        Args:
            config: 配置信息
            
        Returns:
            List[PlanInfo]: 套餐信息列表
        """
        return []
    
    def get_auth_url(self) -> str:
        """返回获取 API Key 的网页地址"""
        return ""
    
    def get_cookie_login_url(self) -> str:
        """返回需要登录的网页（用于自动获取Cookie）"""
        return ""
    
    def validate_config(self, config: dict) -> bool:
        """验证配置是否有效"""
        if self.auth_type == "api_key":
            return bool(config.get("api_key"))
        elif self.auth_type == "cookie":
            return bool(config.get("cookie"))
        return False
```

## 九、注意事项

### 9.1 Cookie 获取方式
- 使用 Playwright 打开浏览器
- 用户手动登录
- 程序自动获取 Cookie
- Cookie 保存到本地配置文件
- Cookie 过期后提示重新获取

### 9.2 免费版限制逻辑
- 启动时检查 License
- 免费版最多启用 3 个厂商
- 超出限制时提示升级
- Plan 监控功能仅限专业版

### 9.3 错误处理
- 网络错误：显示错误信息，不影响其他厂商
- Cookie 过期：提示重新获取
- API Key 无效：提示重新配置
- 接口变更：记录日志，显示"暂不支持"

### 9.4 后续扩展
- 支持更多厂商
- 历史记录功能
- 导出报表
- 多语言支持
